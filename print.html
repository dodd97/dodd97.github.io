1. 文件结构

  • 一个标准的Linux版本调度软件文件夹,包含以下内容

  1. wwwroot文件夹

    • 前端文件夹

  2. Path文件夹 或 Path.json文件

    • 路径文件/文件夹
    • 有的项目表现为 路径文件夹, 如组件项目
    • 有的项目表现为 路径文件, 如包装线项目
    • 有的项目没有, 如舟房

  3. Log文件夹

    • 日志文件夹, 由软件运行后自动生成
    • 不会出现在提供的初版或更新版软件包里

  4. DataBase文件夹

    • 数据库文件夹, 存放数据库文件
    • 数据库文件一般名为 JnRobotManager.db
    • 一般情况下, 数据库文件夹只会出现在初版的软件包里或者数据库结构有变化的更新版里

  5. JunionSeerRobotManager-M3

    • 软件更新的主体, 不同项目名字不一样
    • Linux版本一般表现为没有后缀名
    • Windows版本一般后缀名为exe

  6. *.so

    • 运行的依赖文件, 必须

  7. *.pdb

    • 开发调试文件

    • 无用,可删

  8. appsetting.json

    • 软件配置文件

    • 如果没有特殊说明,软件配置文件无需更新,优先使用生产现场的版本

  9. nlog.config

  • 日志配置文件

  • 如果没有特殊说明,日志配置文件无需更新

忻州一道_舟房

[TOC]

1 倒舟

1.1 概述

  1. 倒舟机结构
左抓舟左暂存左背负
机械臂
右抓舟右暂存右背负

  1. 作用: 将携带舟的agv在抓舟agv和潜伏(背负)agv之间切换

  2. 定义:

    • 换舟: 就是倒舟

    • 倒舟需求: 一次倒舟行为

    • 倒舟序列: 所有的倒舟需求,包括web页面-换舟序列

    • 暂存位: 本来是缓存位,存放一套舟的地方, 现在改完满换满的暂存位

    • 虚拟货架: 功能实现中使用的不存在的库位,仅存在与数据层面

  3. 倒舟需求的类型:

    1. 满换满: 抓舟口一套舟A,背负口一套舟B,当前需求完成后,A转移到背负口被放到货架上, B转移到抓舟口被抓舟agv接走
    2. 单上: 背负口一套舟, 转移到抓舟口被抓舟agv接走,背负口留下空货架
    3. 单下: 抓舟口一套舟, 转移到背负口的货架上

  4. 设定

    1. 倒舟分左右两个方向
    2. 每个方向只会存在一个未完成或未取消的需求
    3. 如果当前没有正在执行的需求,但是两个方向都存在未开始的任务,会选择优先级更高的先执行,而不是按创建时间
    4. 优先级: 满换满 > 单下 > 单上

1.2 实现逻辑

1.2.1 整体逻辑

flowchart LR
	A[需求创建] --> B[倒舟开始] --> C[前置任务] --> D[机械臂搬运] --> E[后置任务] -->F[倒舟结束]

1.2.2 需求创建

  1. 需求创建每个方向单独计算,下面说的agv都是同一个方向的.

  2. 关联数据

    • agvToCatch: 带着下线舟的抓舟agv/从镀膜机接完料的agv/准备送去倒舟机抓舟口的agv(数量)
    • agvFromCatch: 带着上线舟的抓舟agv/从倒舟机抓舟口接完料的agv(数量)
    • storageToCatch: 一个带着待倒上线舟的库位, 允许不存在
    • storageToCatchEmpty: 一个可以送去倒舟机背负口的空货架, 允许不存在
    • agvCache: 设置的预留抓舟agv/ 提前倒好准备被镀膜机叫送料的agv(数量)

  3. 逻辑判断

    if (agvToCatch > 0) {
	// 存在带着下线舟的agv
	if (存在storageToCatch 且 agvCache > agvFromCatch) {
		// 库位这边有上线舟可以换 且 预留agv数量没满
		创建满换满倒舟需求
	} else if (存在storageToCatchEmpty) {
		// 库位有空货架可以接倒下来的下线舟
		创建单下需求
	}
} else if (agvCache > agvFromCatch 且 存在storageToCatch) {
	// 预留agv数量没满 且 库位有上线舟可以倒
	创建单上需求
}

2 双上下

  • 需求: 现场要求部分料口需要在满足可以连续两次送料的情况下才接受任务响应.并且连续执行两次任务.

  • 设置: web端-系统设置tab-需要双上下的料口

    • 写入料口id. 多个时, 使用英文逗号隔开

  • 实现逻辑:

    • 设备第一次呼叫任务

      • 当不存在符合要求的两个库位,则不响应设备呼叫;
      • 当存在符合要求的两个库位HC1与HC2,
        • 同时锁住两个HC1与HC2, 并且在HC2的占用状态字段里标记 料口id|呼叫类型
        • 创建设备与HC1的相关任务并执行

    • 设备第二次呼叫任务

      • 直接查询占用状态字段为 料口id|呼叫类型库位HC2
        • 创建设备与HC1的相关任务并执行
        • 注:  第二次任务创建的时候HC2的占用状态字段就已经被清空了

public interface IFlyBehavior
{
    // 抽象方法
    public abstract void fly();
}

// makeSound行为接口类
public interface IMakeSoundBehavior
{
    public abstract void makeSound();
}

    • 抽象父类

    // 抽象父类
public abstract class Duck
{
    // 行为接口对象
    public IFlyBehavior flyBehavior;
    public IMakeSoundBehavior makeSoundBehavior;

    // 通用抽象方法
    public abstract void display();
    
    public void swim()
    {
        Console.WriteLine("是鸭子就能游");
    }

    // 父类把方法委托给行为接口对象来调用
    public void performFly()
    {
        flyBehavior.fly();
    }
    public void performMakeSound()
    {
        makeSoundBehavior.makeSound();
    }
}
    • 子类
    public class MallardDuck : Duck
{
    public MallardDuck()
    {
        // 初始化时需要对行为类赋值
        flyBehavior = new FlyWithWings();
        makeSoundBehavior = new Quack();
    }

    // 实现父类的通用抽象方法
    public override void display()
    {
        Console.WriteLine("绿头鸭子");
    }
}
public class RubberDuck : Duck
{
    public RubberDuck()
    {
        flyBehavior = new FlyNoWay();
        makeSoundBehavior = new Squeak();
    }
    public override void display()
    {
        Console.WriteLine("橡皮鸭子");
    }
}
public class WoodenDuck : Duck
{
    public WoodenDuck()
    {
        // 初始化时需要对行为类赋值
        flyBehavior = new FlyNoWay();
        makeSoundBehavior = new Mute();
    }
    public override void display()
    {
        Console.WriteLine("木头鸭子");
    }
}
    • 对象新建与行为调用
    Duck mallardDuck = new MallardDuck();
mallardDuck.display();							// 绿头鸭子
mallardDuck.performFly();						// 翅膀飞
mallardDuck.performMakeSound();					// 嘎嘎叫

Duck rubberDuck = new RubberDuck();
rubberDuck.display();							// 橡皮鸭子
rubberDuck.performFly();						// 不能飞
rubberDuck.performMakeSound();					// 吱吱叫

Duck woodenDuck = new WoodenDuck();

woodenDuck.performFly();						// 不能飞
woodenDuck.performMakeSound();					// 不能叫

  • 这种实现方法会让其他类型的对象可以复用fly和makeSound方法,因为行为不再被限制在类里面.也可以直接新增行为,而不修改任何已有的行为类

  • 如果给一只木头鸭增加了一个推进器,让它拥有了飞行的能力.

    }
